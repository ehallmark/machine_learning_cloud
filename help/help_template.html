Filters

Overview

Filters are how we narrow down a search. Filters can be applied in the Filters tab by clicking the Filters drop-down menu and selecting the desired filter. Most attributes have at least one available filter, and these filters can be combined to form more complex searches. Multiple filters can also be applied to the same attribute.

Filter Types

There are many different types of filters, mostly based on the type of attribute being filtered on (numeric, date, text, etc.).

Include Filter

The Include Filter returns results that match a term. This is useful for matching assets by assignee, classification codes, and many other text fields.

Exclude Filter

The Exclude Filter is the negation of the Include Filter. Results matched by the Exclude Filter will be removed from the returned results.

Between Filter

The Between Filter is applicable only to date fields and number fields. This filter only includes assets where the search field is between a specified range. If the Max value is omitted, the filter will only include results greater than or equal to the Min value. If the Min value is omitted, the filter will only include results less than or equal to the Max value.

Prefix Include Filter

The Prefix Include Filter only includes results where the search field starts with the search input. This can be useful for searching subclasses of a CPC class code.

Prefix Exclude Filter

The Prefix Exclude Filter negates the Prefix Include Filter. This can be useful when omitting assets containing a subclass of a particular CPC classification.

Exists Filter

The Exists Filter return results only if the search field is not blank or empty. For example, this could be used to filter out assets that do not have any citations.

Does Not Exist Filter

The Does Not Exist Filter is the negation of the Exists Filter. This filter will only return results where the specified search field does not exist.

Advanced Text Filter

The Advanced Text Filter provides a syntax for more complex text searching. This filter is particularly useful for searching the larger text fields, such as Claim Text or Abstracts. This has also proved useful for searching assignee names.

Regexp Filter

The Regexp Filter provides users who are familiar with Regular Expressions the ability to leverage the syntax for text searching. Unless you are already familiar with Regexp, we suggest trying the Advanced Text Filter first.

Asset Deduplication Filter

The Asset Deduplication Filter uses the available information from the USPTO to remove counterparts of assets. This can be useful when the user does not wish to return more than one asset in any particular asset family.

Nested Filter

The Nested Filter allows filtering on nested attributes. This filter will include various other filters based on the available sub-attributes.

Acclaim Expert Filter

The Acclaim Expert Filter provides compatibility with the Acclaim Patent Search tool. With a few exceptions, Acclaim Expert Search queries can be copied and pasted back and forth between platforms.

Include with Related Assets Filter

This filter builds off the Include Filter for Asset Numbers, but additionally will return results matching a related asset of the search input.

Exclude with Related Assets Filter

This filter builds off the Exclude Filter for Asset Numbers, but additionally will omit results matching a related asset of the search input.
