Getting Started

Introduction

The AI Search Platform is a patent search tool optimized for advanced search and analytics.

Key Features include

  • Asset Valuation Model
  • Asset Clustering
  • Asset Technology Tagging
  • Advanced Text Querying
  • Arbitrary Text Similarity
  • Patent Similarity
  • Assignee Similarity
  • CPC Class Code Similarity
  • Assignee Normalization
  • Save and Share Search Templates
  • Save and Share Datasets
  • Acclaim Expert Syntax Compatible
  • Generate Advanced Charts and Tables

The system currently contains US patents issued in the past 25 years and US applications published since 2005. Patent data includes USPTO Assignment data, PAIR application data, CPC Classification data, Maintenance Event data, and other data sources. The system is updated on a weekly basis, usually on Saturday mornings. During these updates, new data is ingested and integrated into the system. To speed up this process, the user interface is shutdown while the system is updating. Updates typically take several hours to complete. The inclusion of more patent collections and patent data is expected in future releases.

Attributes

Overview

Attributes are data fields stored for each asset. Examples of attributes include Invention Title, Assignee Name, and Inventor Country. Attributes can be applied in filters, sorted on, used in charts and tables, and exported in spreadsheets. The attributes selected in the Attributes tab of the search form determines which attributes will be returned in search results. Please see the following section for a description of each attribute.

Attribute List

AI Value

AI Value is an asset valuation metric developed internally by GTT Group to identify assets with a higher chance of value. The AI Value takes on values from 0 to 100, and can intuitively be interpreted as the percent probability that an asset is valuable.

Filters

Overview

Filters are how we narrow down a search. Filters can be applied in the Filters tab by clicking the Filters drop-down menu and selecting the desired filter. Most attributes have at least one available filter, and these filters can be combined to form more complex searches. Multiple filters can also be applied to the same attribute.

Filter Types

There are many different types of filters, mostly based on the type of attribute being filtered on (numeric, date, text, etc.).

Include Filter

The Include Filter returns results that match a term. This is useful for matching assets by assignee, classification codes, and many other text fields.

Exclude Filter

The Exclude Filter is the negation of the Include Filter. Results matched by the Exclude Filter will be removed from the returned results.

Between Filter

The Between Filter is applicable only to date fields and number fields. This filter only includes assets where the search field is between a specified range. If the Max value is omitted, the filter will only include results greater than or equal to the Min value. If the Min value is omitted, the filter will only include results less than or equal to the Max value.

Prefix Include Filter

The Prefix Include Filter only includes results where the search field starts with the search input. This can be useful for searching subclasses of a CPC class code.

Prefix Exclude Filter

The Prefix Exclude Filter negates the Prefix Include Filter. This can be useful when omitting assets containing a subclass of a particular CPC classification.

Exists Filter

The Exists Filter return results only if the search field is not blank or empty. For example, this could be used to filter out assets that do not have any citations.

Does Not Exist Filter

The Does Not Exist Filter is the negation of the Exists Filter. This filter will only return results where the specified search field does not exist.

Advanced Text Filter

The Advanced Text Filter provides a syntax for more complex text searching. This filter is particularly useful for searching the larger text fields, such as Claim Text or Abstracts. This has also proved useful for searching assignee names.

Regexp Filter

The Regexp Filter provides users who are familiar with Regular Expressions the ability to leverage the syntax for text searching. Unless you are already familiar with Regexp, we suggest trying the Advanced Text Filter first.

Similarity Filters

Similarity Filters provide access to the platform's internal deep-learning based similarity models. Current implementations include:

  • Asset Similarity
  • Assignee Similarity
  • CPC Code Similarity
  • Dataset Similarity
  • Text Similarity
Asset Deduplication Filter

The Asset Deduplication Filter uses the available information from the USPTO to remove counterparts of assets. This can be useful when the user does not wish to return more than one asset in any particular asset family.

Nested Filter

The Nested Filter allows filtering on nested attributes. This filter will include various other filters based on the available sub-attributes.

Acclaim Expert Filter

The Acclaim Expert Filter provides compatibility with the Acclaim Patent Search tool. With a few exceptions, Acclaim Expert Search queries can be copied and pasted back and forth between platforms.

Include with Related Assets Filter

This filter builds off the Include Filter for Asset Numbers, but additionally will return results matching a related asset of the search input.

Exclude with Related Assets Filter

This filter builds off the Exclude Filter for Asset Numbers, but additionally will omit results matching a related asset of the search input.

Sort/Limit

Sorting

Overview

Sorting is an important tool for displaying the best results first. Our platform allows many sort options, including the ability to sort by any numeric or date attribute, and more.

Overall Score

Sort by Overall Score when using text based queries to provide better rankings of assets based on text search algorithms. The following text attributes are effected: Invention Title, Claim Text, Abstract Text. In addition, if a similarity filter is also used, then the Overall Score will also incorporate the similarity score.

Similarity Score

Sort by Similarity score when using text based queries to provide better rankings of assets based on text search algorithms. Be careful, sorting by Similarity Score without providing a similarity filter will result in an exception. Usually, it is preferred to use the Overall Score for sorting.

No Sort

When no sorting is desired, there is an option for No Sort, which will omit sorting. This option can be useful to speed up searching of large collections.

Random Sort

In some cases when the number of matched results is large and no sorting is desired, it may make sense to randomly sort the possible results and apply a search limit to quickly obtain a sample of the desired results. This method is slower than choosing the No Sort option, but provides a better random selection.

Sort Limit

In the Sorting tab, there is an option to specify the Sort Limit. This limits the maximum number of results that the system will return. Please note there is a soft limit of 10,000, and a hard limit of 100,000. This is a consequence of the high computation cost required to sort large numbers of documents. You are allowed to search for more than the soft limit (10,000), but the search will be significantly slower. Searching for more than the hard limit (100,000) is not supported.

Sort Direction

In the Sorting tab, one can elect to sort in Descending (default) or Ascending order.

Templates

Overview

Templates allow a user to save forms to use later. This saves the user time recreating forms. Templates can also be shared with users in your user group. The Default Templates provide a good starting point for creating your own customized templates.

Datasets

Overview

Datasets allow a user to save an asset list into the system. You can then reference the datasets you create later on when searching, generating tables, or creating charts.

Importing

Datasets can be created in one of two ways.

  1. Create a dataset from the results of the last search.
    Right click on a dataset folder and click Create Dataset -> From Last Generated Report
  2. Create a dataset from a list. Right click on a dataset folder and click Create Dataset -> From Asset List. Then paste an asset list into the prompted text box.
Once created, you may choose a name for the new dataset.

Clustering

Overview

Clustering is an algorithm for grouping together similar assets in a Dataset. This can be useful for identifying natural subsets in a portfolio. In order to perform a clustering on a dataset, right click on the dataset and click Cluster this Dataset. There is currently no limit on the number of assets that can be clustered; however, clustering a very large dataset into a large number of clusters could take several minutes, so please be aware of the time commitment.

Number of Clusters

When clustering a dataset, a prompt will ask you to enter a desired number of clusters or leave blank to determine an optimal number for you. If you already have a rough sense of how many clusters you want, it can be faster to enter in that number directly, otherwise it is usually fine to leave the number blank and let the algorithm automatically determine the number of clusters.

Column Combination Table

Column Combination Tables are designed to group and aggregate data of a single attribute. To group and aggregate data for two attributes, please see the section on Pivot Tables.

Column Combination Count Table

Column Combination Count Tables contain a main attribute and a collecting attribute. The main attribute determines the groups, and the collecting attribute determines the attribute being counted for each group. For example, to count the number of assets for the assignees in a search, the grouping attribute would be the assignee name, and the collecting attribute would be the asset number.

Column Combination Function Table

Column Combination Function Tables are similar to Column Combination Count Tables, but instead of counting the collecting attribute for each group, you can select a collecting function to apply. For example, since the AI Value attribute represents a probability, you could estimate the expected number of valuable assets in each main attribute group by collecting the sum of AI Values.

Available collector functions are:

  • Sum
  • Average (mean)
  • Max
  • Min

Pivot Table

Pivot Tables are Column Combination Tables with an added dimension. Please see the section on Column Combination Tables before reading about Pivot Tables. This allows you to group data based on the intersection of two main grouping attributes. You can think of the resulting table as a two dimensional matrix or grid. For example, a Pivot Table with CPC Code and Assignee Name as the two grouping attributes would produce a table with CPC Codes one one axis and Assignee Names on the other. The actual table values would be calculated for each group based on the collecting attribute and collector function (or count function for Pivot Count Tables).
